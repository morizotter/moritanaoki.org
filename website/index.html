TableView and Data

It’s not big issue. It’s an uninteresting daily topic.

How do you handle table view’s data? Before, I handle it with NSArray or NSMutableArray in the UIViewController’s subclass. At that time, I made the contens of the array in it. Messy… And then, I split data from ViewController, it was OK. But now, I make the data store class for handling the section and row data. The good points are,

  1. You can split model from view controller. ViewController becomes clean!
  2. You can easily make many kinds of cells based on the row data class’s information.
  3. The data store class can cache the data.
